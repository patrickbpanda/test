Assumptions / Methodology Notes
Period: Mon 2026-05-11 → Sun 2026-05-17 (7-day calendar week). Generated as-of Mon 2026-05-11 morning review.
UKG hours: Detailed Calculated Time export covering 2026-05-04 → 2026-05-17 (KUR Retailers LLC · KUR/Retail); current cycle uses Mon–Sun 5/11–5/17 slice. PTO and salary employees identified separately.
UKG rates: Detailed Rates report effective 2026-04-27 → 2026-05-11, joined by employee full name. 100% match rate this store (load_store_data).
Dutchie data: Pipeline run 2026-05-17 23:14 UTC + GAP_BACKFILL pull 2026-05-18 13:24 UTC. /reporting/transactions endpoint stable; aggregator deduplicates by transactionId across W20 + GAP_BACKFILL chunks. Line-item-level rows (items[] array) not pulled by current pipeline — see §5g for ticket detail.
Cross-store rule: When an employee is UKG-assigned to Store A but rings at Store B, both hours AND sales roll to UKG home store (Store A). Both dashboards log the cross-store event as a §5g flag.
Salaried employees: Employees absent from UKG hourly export but present in Dutchie sales (e.g., Ernesto Rivera at Northampton) are salaried, not hourly — sales count toward store totals; $/labor-hr productivity is shown as "n/a — salary" rather than flagged as a gap.
Competitor data: Per CFO directive, pricing data comes from each store's own website or Dutchie direct embed — no Leafly.
